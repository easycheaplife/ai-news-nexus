# 设计规范：跨源互证与话题共振 (Topic Resonance)

- **主题**：实现多平台资讯的语义聚合，展示同一话题在不同平台的共振效应。
- **日期**：2026-05-21
- **状态**：已完成

## 1. 概述
在海量资讯中，单一平台的视角往往片面。本功能通过 AI 语义聚类技术，将讨论同一核心事件的不同平台（Twitter, Reddit, GitHub 等）资讯聚合在一起，形成“话题簇 (Topic Cluster)”，帮助用户全方位审视情报。

## 2. 核心特征

### 2.1 共振指数 (Resonance Score)
- **定义**：基于参与讨论的平台数量、账号权重以及讨论热度计算出的综合得分。
- **目的**：筛选出真正的“全网热点”，过滤局部噪音。

### 2.2 共振卡片 (ResonanceCard.vue)
- **UI**：在主流顶部（简报下方）展示横向排布或网格排布的聚合卡片。
- **视觉**：
    - **平台矩阵**：展示参与讨论的平台图标及其对应的色彩。
    - **观点摘要**：展示该话题的核心共识。
    - **互证链路**：点击可查看该话题下所有来源的详细列表。

### 2.3 语义聚类引擎
- **逻辑**：
    - 使用 Gemini 对资讯进行向量化或实体提取。
    - 后端定期执行聚类算法，将相似度高的内容归入同一 `cluster_id`。
    - 自动生成话题标题和综述。

## 3. 技术实现

### 3.1 数据库架构 (Migrations)
- 引入 `topic_clusters` 表存储话题元数据（标题、综述、共振分）。
- 引入 `cluster_news_mapping` 表实现多对多关联，并定义资讯在话题中的“角色”（如：爆料者、评论者、深度分析）。

### 3.2 后端接口 (Clusters API)
- `GET /clusters/trending`：获取当前热门话题簇。
- `GET /clusters/{id}/news`：获取属于该话题的所有资讯项。

### 3.3 前端集成
- **App.vue**：引入 `ResonanceCard` 组件，在非搜索模式下优先展示热门话题。
- **数据流**：资讯列表现在支持通过 `cluster_id` 进行过滤展示。

## 4. 验收标准
- [x] 系统能自动识别出同一事件在 Twitter 和 Reddit 上的讨论并将其关联。
- [x] 前端能直观看到一个话题由哪些平台“共振”产生。
- [x] 共振分能准确反映话题的全球影响力。
