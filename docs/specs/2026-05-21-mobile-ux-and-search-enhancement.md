# 设计规范：移动端体验优化与增强搜索

- **主题**：提升移动设备上的操作效率，扩展全局搜索的维度。
- **日期**：2026-05-21
- **状态**：已完成

## 1. 概述
随着移动端用户增加，原有的桌面优先布局在小屏设备上显得拥挤。本次更新重点解决了移动端的导航效率问题，并增强了后端搜索的灵活性，使其支持作者维度的精确检索。

## 2. 功能详情

### 2.1 移动端抽屉式导航 (Off-canvas Sidebar)
- **UI**：侧边栏在移动端默认隐藏，通过左上角汉堡菜单触发。
- **行为**：从左侧滑出，提供平台过滤和核心导航选项。经过对比实验，最终选择了抽屉式导航而非底部抽屉，以保证与桌面端逻辑的一致性。
- **优化**：增加了半透明遮罩层，点击遮罩或选择项后自动关闭抽屉。

### 2.2 响应式头部与平台选择
- **UI**：重构了移动端 Header 布局。
- **内容**：在移动端将原本平铺的平台过滤器整合进 Header 右侧，使用更节省空间的 Select 下拉框。
- **交互**：针对触屏优化了点击区域，避免了 SVG 图标难以点中的问题。

### 2.3 作者维度搜索 (Author Search)
- **功能**：全局搜索框现在支持通过 `@handle` 或作者名称进行搜索。
- **技术实现**：
    - 后端修改了搜索过滤逻辑，自动识别查询字符串中的 `@` 前缀。
    - 解决了 JSON 字段在 MySQL 不同版本下的搜索兼容性问题（通过 CAST 转换为字符串进行 LIKE 匹配）。

## 3. 技术实现

### 3.1 前端更新
- **App.vue**：
    - 引入 `showMobileMenu` 状态管理侧边栏。
    - 实现了基于 CSS Transform 的滑入滑出动画。
    - 重构了平台选择逻辑，在小屏下自动切换为 `Select` 组件。
- **NewsCard.vue**：修复了在移动端下按钮点击冒泡导致误触发卡片详情的问题。

### 3.2 后端更新
- **FastAPI / SQLAlchemy**：
    - 更新了 `backend/app/api/v1/news.py` 中的搜索接口。
    - 在 SQL 查询中增加了对 `author_handle` 和 `author_name` 的 OR 匹配逻辑。

## 4. 验收标准
- [x] 移动端用户可以顺滑地打开侧边栏。
- [x] 平台切换在手机上不再占据过多垂直空间。
- [x] 输入 `@username` 能准确搜到该作者的所有资讯。
- [x] 解决了旧版 MySQL 对 JSON 搜索不支持的问题。
