# 设计规范：混合式情报探测系统 (社交图谱 + 语义搜索)

- **主题**：打破固定抓取限制，实现信源自动扩张与热点全网追踪。
- **日期**：2026-05-15
- **状态**：草案

## 1. 概述
目前系统依赖于预设的 KOL 列表。本设计引入“发现引擎 (Discovery Engine)”，通过分析现有高分内容，自动“顺藤摸瓜”寻找新专家（方案 A）并全网搜索关键热词（方案 B）。

## 2. 核心功能设计

### 2.1 信号提取层 (Intelligence Extraction)
- **动作**：升级 `GeminiEvaluator`，在分析每条资讯时，额外输出两个字段：
    - `mentioned_users`: 内容中提到的、转发的或互动的潜在高价值账号。
    - `trending_keywords`: 该资讯涉及的 2-3 个核心技术热词。

### 2.2 社交图谱扩张 (The Network Expander - 方案 A)
- **逻辑**：
    1. 收集 `mentioned_users`。
    2. 检查该用户是否已在库中。
    3. 若不在，调用 Gemini 进行“身份画像”：分析其简介和近期推文，判断是否为“AI 领域专家/深度开发者”。
    4. 若通过，存入 `discovered_kol` 表，并自动加入后续的抓取序列。

### 2.3 语义全网探测 (The Trend Hunter - 方案 B)
- **逻辑**：
    1. 提取今日 Top 5 的 `trending_keywords`。
    2. 使用 Twitter 高级搜索或 Google Search API 进行关键词检索。
    3. 过滤条件：仅限过去 24 小时、互动数（Like/RT）> 阈值的内容。
    4. 将搜索结果送入常规 AI 评分流程。

## 3. 技术实现方案

### 3.1 数据库变更
- 新增 `discovery_pool` 表：
    - `type`: 'user' 或 'keyword'
    - `value`: 账号名或关键词
    - `status`: 'pending' (待审核), 'vetted' (已验证), 'rejected' (已忽略)
    - `source_id`: 发现该信号的原始资讯 ID

### 3.2 发现引擎工作流 (`discovery_run.py`)
1. **分析阶段**：从后端拉取近 24 小时的 90+ 分资讯，汇总信号。
2. **扩张阶段**：对新用户执行 Vetting（AI 身份验证）。
3. **探测阶段**：执行关键词搜索并抓取。

## 4. 实施计划
1. **第一阶段：基建**。增加数据库表，修改 Gemini 提示词。
2. **第二阶段：自动挖人**。实现社交图谱的提取与账号自动验证。
3. **第三阶段：全网搜词**。实现基于热词的全球动态探测。

## 5. 验收标准
- [ ] 采集白名单中能自动增加从未手动添加过、但确实极具价值的 AI 专家。
- [ ] 系统能够抓取到非白名单账号发布的、关于当前最火技术（如 MTP）的爆款资讯。
