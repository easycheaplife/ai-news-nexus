# 设计规范：AI 深度洞察与情报存档

- **主题**：将顶部仪表盘升级为持久化的、由 AI 策划的行业简报。
- **日期**：2026-05-15
- **状态**：草案

## 1. 概述
目前，“每日简报”由前端根据硬编码模板即时生成。本设计引入了一个后端驱动的分析阶段，在每次抓取循环结束后运行，总结当天最重要的语义聚类，并将其持久化存储以供后续访问。

## 2. 功能详情

### 2.1 抓取后分析 (情报综合)
- **逻辑**：在所有爬虫结束后，新的 `InsightGenerator` 将：
    1. 查询数据库中过去 24 小时内的所有新闻项。
    2. 按 `cluster_id`（语义聚类 ID）进行分组。
    3. 提取前 5-10 个聚类（按平均分和数量排序）。
    4. 将聚类名称和核心推荐理由发送给 Gemini。
    5. 生成一份专业的“战略简报”（150-300字）。

### 2.2 数据持久化 (`daily_insights` 表)
- **字段**：
    - `id`：主键
    - `date`：简报日期（唯一索引）
    - `content`：AI 生成的 Markdown 格式简报内容。
    - `hot_topics`：当日热门关键词/聚类的 JSON 列表。
    - `stats_json`：关于各平台新闻量的统计元数据。
    - `created_at`：生成时间戳。

### 2.3 前端集成
- `App.vue` 将从 `/insights/latest` 接口获取最新的简报。
- 如果今天尚未生成简报，则回退到“生成中...”状态或当前的前端模板逻辑。

## 3. 技术实现

### 3.1 后端更新
- **数据库**：在 `schema.sql` 中添加 `daily_insights` 表。
- **模型**：创建 `DailyInsight` 模型和对应的 Pydantic Schema。
- **API**：增加 `GET /insights/latest` 和 `POST /insights/`（供爬虫内部使用）。

### 3.2 爬虫更新
- **run.py**：在执行流程末尾添加对 `generate_daily_briefing()` 的调用。
- **utils/ai.py**：在 `GeminiEvaluator` 中添加 `summarize_clusters(data)` 方法。
