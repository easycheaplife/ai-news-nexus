# 设计规范：深度内容提取 (第一阶段)

- **主题**：提升 GitHub、Reddit 和 Hacker News 采集内容的丰富度。
- **日期**：2026-05-15
- **状态**：实施中

## 1. 概述
为了确保聚合平台不只是“标题列表”，我们需要穿透表层信息，抓取具有实际价值的底层正文和社区讨论。第一阶段聚焦于文本类数据源的深度化。

## 2. 增强逻辑

### 2.1 GitHub：README 深度抓取
- **现状**：仅抓取仓库的 Description。
- **增强**：通过 GitHub 原始文件接口 (`raw.githubusercontent.com`) 尝试下载 `README.md`。
- **处理**：截取 README 前 3000 字喂给 Gemini，以便 AI 能准确总结项目的安装、用途和技术架构。

### 2.2 Hacker News：顶级讨论挖掘
- **现状**：仅抓取标题和 URL。
- **增强**：对每一条 AI 相关的 HN 帖子，递归请求其 `kids` 节点。
- **过滤标准**：提取点赞数最高且字符数 > 60 的前 5 条评论。
- **存储**：将评论以 `--- HN Top Comments ---` 形式追加到 `content` 字段。

### 2.3 Reddit：评论质量升级
- **现状**：抓取前 5 条评论。
- **增强**：引入“信息密度”过滤。优先提取包含代码块、链接或长段落的深度回复，忽略“Awesome”、“LOL”等短评。
- **规模**：提升至前 10 条高质量评论。

## 3. 实施计划

1. **GitHub Scraper 更新**：修改 `github.py`，增加 README 下载逻辑。
2. **HN Scraper 更新**：修改 `hn.py`，实现评论递归抓取。
3. **Reddit Scraper 优化**：调整 `reddit.py` 的评论排序与过滤逻辑。

## 4. 验收标准
- [ ] 所有的 GitHub 项目资讯都包含详细的功能介绍。
- [ ] 热门 HN 帖子下能看到专业人士的深度技术评价。
- [ ] 存入数据库的 `content` 字段平均字数显著提升，AI 总结的精度更高。
