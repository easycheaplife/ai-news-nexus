# 微信公众号抓取 (WeChat Scraper) 设计文档

## 1. 背景与目标
微信公众号是中国 AI 行业深度内容的最核心来源。本设计旨在建立一个稳定的“公众号观察哨”，每日监控并抓取国内顶尖 AI 媒体、实验室和独立观察者的深度长文，并通过 AI 进行全文解析。

## 2. 抓取策略 (Hybrid Extraction)
由于微信公众号的直接爬取难度极高且容易封禁，本设计采用“雷达扫描 + 穿透读取”的混合方案。

### 2.1 更新监控 (Monitoring)
- **方案**: 使用公共的 WeChat-to-RSS 桥接服务（如 `rsshub.app` 或自建实例）。
- **逻辑**: 定时轮询预设的公众号 ID 列表。

### 2.2 全文穿透 (Full Content Extraction)
- **挑战**: 公众号的原始 HTML 结构复杂，且包含大量防盗链图片和脚本。
- **解决方案**: 使用 `https://r.jina.ai/` 服务。
- **执行流程**: 
    1. 从 RSS 中获取文章原始链接。
    2. 将链接传递给 `r.jina.ai` 获取纯净的 Markdown/Text 全文。
    3. 绕过排版干扰，直接获取核心文本。

## 4. 架构设计
- **新增模块**: `scrapers/engines/wechat.py`。
- **游标管理**: 使用文章的 `published` 时间戳作为游标，防止重复。
- **AI 评估**: 
    - 针对长文，AI 会接收 `Full Text`。
    - **Prompt 优化**: 引导 AI 重点提炼“国内本土化进展”、“技术架构突破”或“行业落地案例”。

## 5. 预期产出
- `run.py` 运行时，会显示 `Starting wechat engine...`。
- 系统能抓取到几千字以上的深度长文，并提炼出 5 条以上的硬核 Takeaways。
- 自动识别文中的人物提及，补充国内信源发现池。
